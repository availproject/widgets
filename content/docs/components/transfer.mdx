## Installation

```bash
npx shadcn@latest add @nexus-elements/nexus-one
```

## Usage

Wrap your app with the Nexus provider first. See [Get Started](/docs/get-started) for provider setup.

```tsx showLineNumbers
import { NexusOne } from "@/components/nexus-one/nexus-one";

const USDC_BASE = "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913";
const RECIPIENT = "0xF3a15b38e63dBb1a1b2d7842CcD9B9dD8fB9b2E";

export function SendExample({ address }: { address?: `0x${string}` }) {
  return (
    <NexusOne
      connectedAddress={address}
      config={{
        mode: "send",
        prefill: {
          token: USDC_BASE,
          chain: 8453,
          amount: "0.1",
          recipient: RECIPIENT,
        },
      }}
    />
  );
}
```

## Configuration

Send uses only `mode: "send"`. This mode is exact-out: users enter the token amount to send, pay-with sources are selected, and the flow submits through `swapAndTransfer`.

| Prop | Required | Description |
| --- | --- | --- |
| `config.mode` | Yes | Must be `"send"`. |
| `config.prefill.token` | Optional | Preselects the token to send. |
| `config.prefill.chain` | Optional | Preselects the destination chain. |
| `config.prefill.amount` | Optional | Prefills the exact output send amount. |
| `config.prefill.recipient` | Optional | Prefills the recipient address. A recipient is required before users can submit. |
| `config.allowedSourcePairs` | Optional | Restricts pay-with source assets. |
| `config.allowedDestinationPairs` | Optional | Restricts destination token and chain choices. |
| `connectedAddress` | Optional | Wallet address to use. If omitted, the connected wagmi account is used. |
| `embed` | Optional | Defaults to `true`. Set `false` to render as a modal surface. |
| `onStart`, `onComplete`, `onError`, `onClose` | Optional | Lifecycle callbacks for host apps. |
