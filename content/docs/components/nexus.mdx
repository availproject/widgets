## Configuration

`config.mode` is required and must be `"swap"`, `"send"`, or `"deposit"`.

### Props

| Prop | Type | Notes |
| --- | --- | --- |
| `config` | `object` | Selects the workflow and any mode-specific behavior. |
| `connectedAddress` | `` `0x${string}` `` | Wallet address to use. If omitted, Nexus Widget falls back to the connected wagmi wallet account. |
| `embed` | `boolean` | Defaults to `true`. Set `false` when rendering Nexus Widget as a modal surface. |
| `open`, `onOpenChange`, `defaultOpen` | modal controls | Control modal state when `embed={false}`. |
| `onComplete` | `(explorerUrl?: string) => void` | Called after a successful swap, send, or deposit. |
| `onStart` | `() => void` | Called when the user starts execution. |
| `onError` | `(message: string) => void` | Called when execution fails. |
| `onClose` | `() => void` | Used by modal mode. |
| `onConnectClick` | `() => void \| Promise<void>` | Called when the internal Connect Wallet CTA is clicked. Use this to open your app/header wallet modal. |
| `onConnectWallet` | `() => void \| Promise<void>` | Backward-compatible connect callback prop. |

### Shared Config

| Field | Modes | Notes |
| --- | --- | --- |
| `destination.chain` | all | Required for deposit. Optional for send/swap; when supplied, destination chain selection is restricted. |
| `destination.tokens` | all | Required for deposit with at least one token. Optional for send/swap; when supplied with `destination.chain`, destination token selection is restricted. |
| `recipientAddress` | send/swap | Prefills the recipient. In send mode, a supplied recipient is locked. |
| `prefill.amount` | deposit/send | Optional amount prefill. Must be greater than `0`. |
| `prefill.token` | send/swap | Optional initial destination/receive token: `{ chain, address, symbol, decimals, logo? }`. It does not restrict later token choices. |
| `validation.minAmount` | deposit/send | Optional minimum amount. Must be `0` or greater. |
| `validation.maxAmount` | deposit/send | Optional maximum amount. Must be greater than `0`. |
| `appearance.heading` | all | Overrides the idle screen heading. |
| `appearance.appName` | all | App/protocol name used in deposit progress and success UI. |
| `appearance.appLogoURL` | all | App/protocol logo URL. Must start with `http://` or `https://`. |
| `appearance.mode` | all | `"system"`, `"light"`, or `"dark"`. |
| `appearance.primaryColor` | all | Any valid browser-supported CSS color. Text color is chosen automatically for contrast. |

Swap supports `prefill.token` only. It does not support amount prefill or validation yet.

Use `destination.tokens` when you want to restrict the destination/receive selector. Use `prefill.token` when you only want an initial token selected and still want users to choose from the full supported list. If both are supplied, `destination.tokens[0]` is honored and `prefill.token` is ignored.

### Deposit Config

Deposit requires a fixed destination chain, at least one destination token, a deposit contract address, and an execution builder.

```ts
type NexusWidgetDepositConfig = {
  mode: "deposit";
  destination: {
    chain: number;
    tokens: {
      address: `0x${string}`;
      symbol: string;
      decimals: number;
      logo?: string;
    }[];
  };
  depositAddress: `0x${string}`;
  executeDeposit: (
    tokenSymbol: string,
    tokenAddress: `0x${string}`,
    amount: bigint,
    chainId: number,
    user: `0x${string}`,
  ) => {
    to: `0x${string}`;
    data?: `0x${string}`;
    value?: bigint;
    gas?: bigint;
    tokenApproval?: {
      toTokenAddress: `0x${string}`;
      amount: bigint;
      spender: `0x${string}`;
    };
  };
  prefill?: { amount?: string };
  validation?: { minAmount?: string; maxAmount?: string };
  appearance?: NexusWidgetAppearance;
};
```

### Deposit

```tsx showLineNumbers
import { NexusWidget } from "@/components/nexus/nexus";
import { encodeFunctionData } from "viem";

// Configure these for your app.
const APP_DEPOSIT_CONTRACT = "0x...";
const DESTINATION_TOKEN = "0x...";
const DESTINATION_CHAIN_ID = 42161;
const DESTINATION_TOKEN_SYMBOL = "USDT";
const DESTINATION_TOKEN_DECIMALS = 6;

const APP_DEPOSIT_ABI = [
  {
    inputs: [
      { name: "token", type: "address" },
      { name: "amount", type: "uint256" },
      { name: "user", type: "address" },
    ],
    name: "deposit",
    outputs: [],
    stateMutability: "nonpayable",
    type: "function",
  },
] as const;

function buildDepositExecuteConfig(
  tokenAddress: `0x${string}`,
  amount: bigint,
  user: `0x${string}`,
) {
  return {
    to: APP_DEPOSIT_CONTRACT,
    data: encodeFunctionData({
      abi: APP_DEPOSIT_ABI,
      functionName: "deposit",
      args: [tokenAddress, amount, user],
      // Aave reference: supply(asset, amount, onBehalfOf, referralCode)
    }),
    tokenApproval: {
      toTokenAddress: tokenAddress,
      amount,
      spender: APP_DEPOSIT_CONTRACT,
    },
  };
}

const depositConfig = {
  mode: "deposit",
  destination: {
    chain: DESTINATION_CHAIN_ID,
    tokens: [
      {
        address: DESTINATION_TOKEN,
        symbol: DESTINATION_TOKEN_SYMBOL,
        decimals: DESTINATION_TOKEN_DECIMALS,
      },
    ],
  },
  depositAddress: APP_DEPOSIT_CONTRACT,
  executeDeposit: (_symbol, tokenAddress, amount, _chainId, user) =>
    buildDepositExecuteConfig(tokenAddress, amount, user),
  appearance: {
    appName: "Your App",
    heading: "Deposit",
    mode: "system",
  },
} as const;

<NexusWidget config={depositConfig} connectedAddress={address} />;
```
