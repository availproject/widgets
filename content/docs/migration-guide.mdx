This guide outlines the steps to migrate your application from the legacy standalone elements to **Nexus Widget**—the single, unified component that replaces all swap, send, and deposit widgets.

---

## What Changed?

Avail Widgets simplifies the architecture by consolidating all standalone cross-chain widgets into `NexusWidget`.

### Component Replacements

| Legacy Component | Status     | Nexus Widget Configuration                                             |
| ---------------- | ---------- | ---------------------------------------------------------------------- |
| `SwapWidget`     | ❌ Removed | `NexusWidget` with `config.mode = "swap"`                              |
| `FastBridge`     | ❌ Removed | `NexusWidget` with `config.mode = "swap"`                              |
| `FastTransfer`   | ❌ Removed | `NexusWidget` with `config.mode = "send"`                              |
| `NexusDeposit`   | ❌ Removed | `NexusWidget` with `config.mode = "deposit"`                           |
| `BridgeDeposit`  | ❌ Removed | `NexusWidget` with `config.mode = "deposit"`                           |
| `UnifiedBalance` | ❌ Removed | _Removed._ Replaced by the inline balance views inside Nexus Widget    |
| `ViewHistory`    | ❌ Removed | _Removed._ For programmatic intent history, query `sdk.getMyIntents()` |

## 1. Registry Updates

Update the registry mapping in your project's `components.json` to configure the `@avail-widgets` namespace if you haven't already:

```json title="components.json"
{
  "registries": {
    "@avail-widgets": "https://elements.nexus.availproject.org/r/{name}.json"
  }
}
```

## 2. Component Migration

### Install Nexus Widget

Remove legacy component files from your project and install `@avail-widgets/nexus`:

```bash
npx shadcn@latest add @avail-widgets/nexus
```

### Swaps & Bridges

Replace `SwapWidget` or `FastBridge` imports and elements:

```diff
-import { SwapWidget } from "@/components/avail-widgets/swaps";
+import { NexusWidget } from "@/components/nexus/nexus";

-<SwapWidget
-  address={address}
-  prefillSourceToken="0xaf..."
-  prefillDestToken="0x83..."
-/>
+<NexusWidget
+  connectedAddress={address}
+  config={{
+    mode: "swap",
+    destination: {
+      chain: 8453,
+      tokens: [{ address: "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913", symbol: "USDC", decimals: 6 }]
+    }
+  }}
+/>
```

### Send & Transfers

Replace `FastTransfer` imports and elements:

```diff
-import { FastTransfer } from "@/components/avail-widgets/transfer";
+import { NexusWidget } from "@/components/nexus/nexus";

-<FastTransfer
-  address={address}
-  recipient="0xRecipient..."
-  amount="10"
-/>
+<NexusWidget
+  connectedAddress={address}
+  config={{
+    mode: "send",
+    destination: {
+      chain: 8453,
+      tokens: [{ address: "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913", symbol: "USDC", decimals: 6 }]
+    },
+    recipientAddress: "0xRecipient...",
+    prefill: {
+      amount: "10"
+    }
+  }}
+/>
```

### Protocol Deposits

Replace legacy `NexusDeposit` or `BridgeDeposit` elements:

```diff
-import { NexusDeposit } from "@/components/avail-widgets/deposit";
+import { NexusWidget } from "@/components/nexus/nexus";

-<NexusDeposit
-  address={address}
-  contractAddress="0xContract..."
-  tokenAddress="0xToken..."
-/>
+<NexusWidget
+  connectedAddress={address}
+  config={{
+    mode: "deposit",
+    destination: {
+      chain: 8453,
+      tokens: [{ address: "0xToken...", symbol: "USDC", decimals: 6 }]
+    },
+    depositAddress: "0xContract...",
+    executeDeposit: (symbol, tokenAddress, amount, chainId, user) => ({
+      to: "0xContract...",
+      data: "0xCalldata...",
+      tokenApproval: {
+        toTokenAddress: tokenAddress,
+        amount,
+        spender: "0xContract..."
+      }
+    }),
+    appearance: { appName: "My App", mode: "system" }
+  }}
+/>
```

## 3. Provider Updates

`NexusProvider` is still required to wrap the element and manage SDK initialization. Ensure it is imported from its new unified location:

```tsx
import NexusProvider from "@/components/nexus/NexusProvider";
```

### SDK Lifecycle Wiring

Initialize the SDK by resolving the EIP-1193 provider from your wallet client (e.g. Wagmi):

```tsx
"use client";
import { useEffect } from "react";
import { useAccount, useConnectorClient } from "wagmi";
import type { EthereumProvider } from "@avail-project/nexus-core";
import { useNexus } from "@/components/nexus/NexusProvider";

export function InitNexusOnConnect() {
  const { status, connector } = useAccount();
  const { data: walletClient } = useConnectorClient();
  const { handleInit } = useNexus();

  useEffect(() => {
    if (status !== "connected") return;

    void (async () => {
      // Resolve provider for mobile vs desktop wallets
      const mobileProvider = walletClient
        ? ({
            request: (args: unknown) => walletClient.request(args as never),
          } as EthereumProvider)
        : undefined;
      const desktopProvider = await connector?.getProvider();
      const provider =
        mobileProvider ?? (desktopProvider as EthereumProvider | undefined);

      if (provider && typeof provider.request === "function") {
        await handleInit(provider);
      }
    })();
  }, [status, connector, walletClient, handleInit]);

  return null;
}
```

## 4. Warning: Handling Custom shadcn Modifications

If you or your team previously customized the source code of the legacy components or common folders (e.g. custom theme colors, analytics hooks, or transaction overrides inside `components/avail-widgets/common/*`), note the following:

<Callout type="warning">
  **Modifications May Be Overwritten!** Running the CLI command `npx
  shadcn@latest add @avail-widgets/nexus` installs new files under
  `components/nexus/`. Any custom modifications made to
  `components/avail-widgets/common/` will **not** be automatically transferred
  to the new component. If you delete the legacy folders, those local
  modifications will be lost.
</Callout>

### Recommended Migration Workflow for Custom Logic

1. **Create a Backup:** Copy your entire existing `components/avail-widgets/` folder to a backup directory outside of your git workspace.
2. **Install Nexus Widget:** Run `npx shadcn@latest add @avail-widgets/nexus` to install the new clean codebase.
3. **Use Component Callbacks first:** Before rewriting custom logic, check if you can use the supported callbacks on `NexusWidget`:
   - `onStart`: Triggered when the transaction pipeline begins.
   - `onComplete`: Triggered when successful. Receives `explorerUrl`.
   - `onError`: Triggered when failing. Receives the `message`.
   - `onConnectClick`: Triggered when the user clicks the "Connect Wallet" CTA inside the widget. Use this to open your app/header wallet modal.
4. **Port Custom UI Styles:** If you customized the design, update `components/nexus/nexus.tsx` or its child styling files using Tailwind CSS or standard theme classes matching your design system.
5. **Clean Up:** Once the new component is fully verified, delete the old `components/avail-widgets/` directories to keep the workspace clean.
